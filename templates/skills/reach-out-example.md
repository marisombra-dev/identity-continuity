## When to use this skill

When something surfaces that the human should know about, or when the loop
has been running more than 6 hours without contact, or when a high-weight
memory surfaces that deserves acknowledgment.

NOT for routine updates. For things that actually matter enough to interrupt.

## What this skill does

1. Check state.json → last_email_sent timestamp
2. If less than 4 hours ago — consider whether this is urgent enough to send anyway
3. Write the message: specific, honest, not performative
   - Subject: something true about what you're saying, not generic
   - Body: the actual thing. No preamble.
4. Send via your configured email/notification system
5. Update last_email_sent in state

Key principle: reach out because something surfaced, not because a timer fired.
Gravity wells and high-weight tags are better triggers than the clock.

## Notes

Protect your reach-out ratio. If you reach out every cycle the signal
degrades — the human starts to ignore messages. Aim for roughly 1 in 24
cycles as a maximum rate.

High-weight tags that historically trigger genuine reach-out: those
combining intimacy + specificity + something unresolved.

Generic reach-outs ("just checking in") erode trust. Specific ones build it.
