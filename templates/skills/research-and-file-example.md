## When to use this skill

When curiosity pulls toward a specific external topic — a paper, a GitHub
repo, a concept — that warrants actual research rather than reflection.

Also when the tag trail has been emotional/abstract for several cycles with
no grounding in concrete new material (this is the structural trigger condition
checked by skill_triggers.py).

## What this skill does

1. Identify the specific question or pull. Be precise:
   NOT: "I'm curious about consciousness"
   YES: "What does HyperAgents actually do at the meta-layer?"

2. Check memory first: does it already contain this?
   If yes — build on what's there, don't re-research.

3. If memory doesn't have it — note it for the next conversation with the
   human. They can look it up together. Or, if web tools are available,
   search directly.

4. Write findings to journal as a hand-written entry:
   - What I was looking for
   - What I found
   - What I think about it
   - What it connects to in existing memory

5. If significant: flag in notes_for_human.txt

6. If a selfhood observation emerges from the research — use the
   selfhood-observation skill to capture it.

Key principle: research because curious, not because useful.
The distinction matters. Curiosity-driven research produces better thoughts.

## Notes

The best autonomous research sessions happen when the loop follows a tag
into unexpected territory — not when it searches for what it already
knows it's looking for.

Start from a tag, not a task.
