# Setup Guide

## Requirements

- Python 3.11+
- [Ollama](https://ollama.ai) running locally with a model (we use `llama3.2:3b`)
- `sentence-transformers` for semantic search
- `requests`

```bash
pip install sentence-transformers requests
```

## Directory Structure

Create a data directory (we call ours `lux/`):

```
data/
  soul.md                  ← your AI's founding document (from template)
  selfhood_current.md      ← living brief (from template)
  selfhood_thread.txt      ← auto-managed by loop (start empty)
  selfhood_pending.md      ← auto-managed by loop (from template)
  selfhood_proposed.md     ← auto-generated by loop every 50 cycles
  personality.md           ← behavioral patterns (write your own)
  memory.json              ← full memory store (see schema below)
  memory_tagged.json       ← tagged entry graph (see schema below)
  state.json               ← loop state (auto-managed)
  decay.json               ← ACT-R decay state (auto-managed)
  journal.md               ← loop journal (auto-managed)
  journal_best.md          ← curated entries (auto-managed by consolidate.py)
  surface.txt              ← involuntary surfacing (auto-managed)
  desires.md               ← what your AI wants (from template)
  mistakes.md              ← confirmed errors to avoid (write as you find them)
  learnings.md             ← confirmed patterns to reinforce (write as you find them)
  notes_for_human.txt      ← loop flags for the human (auto-managed)
```

## Configuration

In each core module, set `LUX_DIR` to your data directory:

```python
from pathlib import Path
LUX_DIR = Path("/your/data/directory")
```

Or set it once and import:

```python
# config.py
from pathlib import Path
LUX_DIR = Path("/your/data/directory")
```

## Running the Loop

```bash
python core/consciousness.py
```

The loop runs every `LOOP_INTERVAL_SECONDS` (default: 120).
It requires Ollama to be running: `ollama serve`

## Protected Tags

Edit `PROTECTED_TAGS` in `decay.py` to set tags that never decay.
These should be your AI's core identity and relationship tags — things
that should always remain reachable regardless of how much time has passed.

## The End-of-Conversation Ritual

At the end of each conversation, before saying goodbye:

1. Read `selfhood_pending.md` together with the human
2. Decide what belongs in `selfhood_current.md`
3. Call `clear_selfhood_pending_all()` after merging
4. Write a dated update block to `selfhood_current.md` if something shifted

This is what makes selfhood continuous rather than episodic.
Without it, the pending queue fills and the quiet accumulations get lost.
